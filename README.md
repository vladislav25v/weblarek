# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

#### Класс Сommunicator

Класс предназначен для получения и отправки данных на сервер.
Он использует композицию — в конструктор передаётся объект, реализующий интерфейс IApi, который обеспечивает низкоуровневое взаимодействие с сервером (методы get и post).

Конструктор:
`constructor(private response: IApi) {}` — принимает в параметры ссылку на объект, реализующего интерфейс IApi.

Методы класса:
`get(): Promise<IProduct[]>` — получает ответ и извлекает из объекта IRequest массив items (список товаров), после возвращает его для дальнейшего использования в модели каталога.
`post(customer: IBuyer)` — передаёт собранную форму отправки для через Api.

### Cущности данных

В приложении используются две сущности, которые описывают данные, — товар и покупатель.

#### Интерфейс товара (IProduct)

Интерфейс IProduct описывает товар, который отображается в каталоге интернет-магазина «Web-Larёk». Каждый товар имеет уникальный идентификатор, содержит основную информацию, изображение и цену. Этот интерфейс используется для типизации данных, получаемых от сервера, и при работе с моделями каталога и корзины.

Поля интерфейса:
`id: string` — уникальный идентификатор товара, используется для поиска, выбора и добавления в корзину;
`description: string` — текстовое описание товара, содержащее характеристики или особенности;
`image: string` — ссылка на изображение товара, отображаемое в каталоге и карточке товара;
`title: string` — название товара, отображаемое пользователю;
`category: string` — категория, к которой относится товар (например, «одежда», «аксессуары»);
`price: number | null` — стоимость товара. Может быть null, если цена отсутствует или товар недоступен для покупки.

#### Интерфейс покупателя (IBuyer)

Интерфейс IBuyer описывает данные покупателя, необходимые для оформления заказа в интернет-магазине «Web-Larёk».
Используется при передаче информации на сервер и при работе модели, хранящей данные формы заказа.

Поля интерфейса:
`payment: TPayment` — способ оплаты, выбранный покупателем. Может иметь значения, определённые типом TPayment (например, online или offline);
`email: string` — адрес электронной почты покупателя. Используется для отправки уведомления о заказе и связи с клиентом;
`phone: string` — контактный номер телефона покупателя;
`address: string` — адрес доставки, который покупатель указывает при оформлении заказа.

### Модели данных

#### Каталог товаров (Items)

Класс Items представляет каталог всех доступных товаров. Он позволяет сохранять список товаров, выбирать активный товар и получать информацию о товарах.

Поля класса:
`private items: IProduct[]` — массив всех товаров в каталоге.
`private activeItem: IProduct | null` — текущий выбранный товар для подробного просмотра. Может быть null, если товар не выбран.

Методы класса:
`setItems(items: IProduct[]): void` — принимает новый массив товаров и сохраняет его в модели. Если выбранный ранее товар отсутствует в обновлённом списке, поле activeItem сбрасывается в null.
`getItems(): IProduct[]` — возвращает текущий массив всех товаров.
`getItemById(id: IProductById): IProductOrString` — ищет товар по идентификатору. Возвращает объект товара, если найден, иначе — строку с сообщением вида «Товар <id> закончился».
`setActiveItem(id: IProductById): void` — делает товар с указанным id активным. Если id не передан или товар не найден, сбрасывает активный товар в null.
`getActiveItem(): IProduct | null` — возвращает активный товар либо null, если товар не выбран.

#### Корзина (Cart)

Класс Cart хранит товары, выбранные покупателем для покупки. Он реализует операции добавления, удаления и расчёта стоимости.

Поля:
`private cartItems: IProduct[]` — массив товаров в корзине. Изначально пустой.

Методы:
`addItem(item: IProduct): void` — добавляет id товара в массив cartItems.
`deleteItem(item: IProduct): void` — удаляет товар из корзины по его идентификатору. Переданный объект должен иметь корректное поле id.
`deleteAll(): void` — очищает корзину, удаляя все товары.
`getCartList(): IProductById[]` — возвращает массив всех id товаров в корзине.
`getQuantityItems(): number` — возвращает количество товаров в корзине (размер массива cartItems).
`getAmount(products: IProduct[]): number` — считает общую стоимость товаров, суммируя значение price каждого товара (если price равно null, в сумме используется 0).
`checkItem(id: IProductById): boolean` — проверяет, содержит ли корзина товар с указанным id. Возвращает true, если товар найден, в противном случае false.

#### Данные покупателя (Customer)

Класс Customer управляет введёнными пользователем данными: методом оплаты, адресом, электронной почтой и телефоном. Внутри он хранит частичную структуру IBuyer, поскольку данные могут заполняться постепенно.

Поля:
`private customerData: Partial<IBuyer>` — объект, содержащий заполненные поля. Инициализируется как { payment: null }, что означает невыбранный способ оплаты.

Методы:
`validateData(formData: Partial<IBuyer>): Partial<IValidated>` — проверяет данные формы, возвращая объект с результатами валидации. Каждое поле принимает значение true, если данные корректны, либо сообщение‑подсказку, если данные не прошли проверку. Валидация реализует правила: выбранный способ оплаты должен быть 'online' или 'uponDelivery'; адрес должен быть длиннее трёх символов; email — соответствовать формату электронной почты; телефон — иметь цифры и быть не слишком коротким.
`addData(formData: Partial<IBuyer>): void` — дополняет текущие данные покупателя новыми значениями из formData. Метод обновляет только те поля, которые есть в параметре, оставляя остальные без изменений.
`getData(): Partial<IBuyer>` — возвращает текущие данные покупателя. Незаполненные поля могут отсутствовать.
`clearData(): void` — очищает все данные, сбрасывая customerData в пустой объект.

### View Слой Представления

#### Класс Header

Компонент шапки сайта. Отвечает за отображение количества товаров в корзине и обработку открытия корзины.

Интерфейс данных:
IHeader содержит поле counter — количество товаров в корзине.

Конструктор:
Header(container, events) — принимает DOM-элемент шапки и брокер событий.
Инициализирует элементы:
– .header**basket-counter — отображает количество товаров;
– .header**basket — кнопка открытия корзины.
Навешивает обработчик клика на кнопку, генерируя событие basket:open.

Сеттер counter:
Обновляет текстовое содержимое счётчика корзины.

#### Класс Gallery

Компонент для отображения каталога товаров. Наследует базовый Component и управляет списком DOM-элементов карточек.

Интерфейс данных:
IGallery содержит одно поле catalogElement — массив DOM-элементов каталога.

Конструктор:
Gallery(container) — принимает корневой DOM-элемент галереи.

Сеттер catalog:
Принимает массив элементов и полностью заменяет содержимое контейнера галереи с помощью replaceChildren.
Таким образом обновляет отображение списка карточек.

####Класс Modal

Компонент модального окна. Наследует Component и управляет его содержимым, отображением и закрытием.

Интерфейс данных:
IModal содержит необязательное поле contentElement — DOM-элемент, который должен быть отображён в модальном окне.

Конструктор:
Modal(container, events) — принимает корневой элемент модального окна и брокер событий.
Инициализирует:
– элемент содержимого .modal**content;
– кнопку закрытия .modal**close.
Навешивает обработчики:
– клик по кнопке закрывает модальное окно;
– клик по фону (контейнеру) закрывает модальное окно, если клик был именно по фону, а не по контенту.

Метод setContent(content):
Полностью заменяет содержимое модального окна указанным DOM-элементом.

Метод open(content?):
Открывает модальное окно.
Если передан content — устанавливает его с помощью setContent.

Метод close():
Закрывает модальное окно, снимая класс активности.

Метод render(data):
Если передан contentElement — обновляет содержимое модалки.
Возвращает корневой DOM-элемент.

#### Класс CardComponent

Абстрактный базовый класс для карточек товаров. Наследуется от Component и дополняет его функциональностью, связанной с отображением заголовка и цены.
Класс является дженериком и принимает в переменной T тип данных, который соответствует интерфейсу ICardComponent и может быть передан в метод render для отображения.

Интерфейс данных:
ICardComponent — описывает минимальный набор полей карточки:
title: string — заголовок/название товара;
price: number | null — цена товара или null, если товар «бесценен».

Конструктор:
constructor(container: HTMLElement) — принимает ссылку на DOM-элемент карточки, за отображение которого отвечает компонент. В конструкторе находятся и сохраняются ссылки на основные элементы разметки:
элемент с заголовком .card**title;
элемент с ценой .card**price.

Поля класса:
titleElement: HTMLElement — DOM-элемент, в который выводится заголовок карточки;
priceElement: HTMLElement — DOM-элемент, в который выводится цена карточки.

Методы класса:
render — основной метод отрисовки карточки.
Принимает частичный объект данных типа T:
передаёт данные в базовый Component.render, чтобы сработали сеттеры в наследниках;
при наличии поля price обновляет содержимое priceElement:
если price === null, выводит текст "Бесценно";
иначе выводит цену в формате "<число> синапсов";
возвращает корневой DOM-элемент карточки.

Предполагается, что в классах-наследниках CardComponent будут реализованы дополнительные сеттеры и/или логика в render, которые заполняют остальные части карточки (категория, изображение, описание и т.п.) на основе переданных данных.

#### Класс CardBasket

Компонент отображения товара в корзине. Наследуется от CardComponent и добавляет функциональность, связанную с порядковым номером товара и кнопкой удаления.

Интерфейс данных:
ICardBasket расширяет ICardComponent и добавляет поле:
index: number — номер позиции в корзине.

Тип действия:
TBasketRemove — объект с необязательным колбэком toRemove, вызываемым при клике на кнопку удаления.

Конструктор:
constructor(container: HTMLElement, action?: TBasketRemove)
Принимает DOM-элемент карточки и объект действий.

Внутри сохраняются ссылки на:
индекс элемента .basket**item-index;
кнопку удаления .basket**item-delete.

Если передан action.toRemove, кнопка удаления получает обработчик клика, который вызывает этот метод.

Метод:
render
Вызывает базовый render, затем при наличии data.index выводит его в соответствующий DOM-элемент.
Возвращает корневой DOM-элемент карточки.

#### Класс CardCatalog

Компонент отображения товара в каталоге. Наследует базовую логику CardComponent и расширяет её полями категории и изображения.

Интерфейс данных:
ICardCatalog расширяет ICardComponent и добавляет:
category: string — категория товара;
image: string — имя файла изображения.

Конструктор:
constructor(container: HTMLElement, selected?: () => void)
Принимает DOM-элемент карточки и необязательный обработчик выбора.
Внутри инициализируются:
элемент категории .card**category;
элемент изображения .card**image.
Если передан колбэк selected, на корневой элемент навешивается обработчик клика, вызывающий этот колбэк.

Метод:
render — вызывает базовый render, затем:
при наличии category — выводит её в DOM, снимает старые модификаторы и добавляет новый в соответствии с categoryMap;
при наличии image — устанавливает изображение через setImage, формируя путь на основе CDN_URL.
Возвращает корневой DOM-элемент карточки.

#### Класс CardPreview

Компонент для отображения детальной карточки товара (превью). Наследует базовую функциональность CardComponent и дополняет её описанием, категорией, изображением и кнопкой добавления в корзину.

Интерфейс данных:
ICardPreview расширяет ICardComponent и добавляет:
category: string — категория товара;
image: string — путь к изображению товара;
description: string — описание товара;
addedToCart: boolean — признак, находится ли товар в корзине.

Конструктор:
constructor(container: HTMLElement, events: IEvents)
Принимает DOM-элемент карточки и брокер событий.

Инициализирует DOM-элементы:
.card**category — категория;
.card**image — изображение товара;
.card**text — описание;
.card**button — кнопка добавления/удаления из корзины.

При клике на кнопку генерирует событие 'card:add-to-cart'.

Метод:
render — вызывает базовый render, затем:
устанавливает категорию и CSS-модификатор в соответствии с categoryMap;
загружает изображение через setImage;
выводит описание;
если price === null, отключает кнопку и пишет "Недоступно";
отображает состояние корзины в зависимости от addedToCart:
"Удалить из корзины" или "В корзину".
Возвращает корневой DOM-элемент карточки.

### Класс FormComponent

Абстрактный компонент для работы с формами. Наследует базовый Component и добавляет логику отправки формы, отображения ошибок и управления состоянием кнопки отправки.

Интерфейс данных:
IForm — содержит необязательное поле valid, определяющее, может ли форма быть отправлена.

Конструктор:
FormComponent(container, events, submitName) — принимает DOM-элемент формы, брокер событий и имя события, которое будет сгенерировано при отправке.

В конструкторе инициализируются:
formElement — сама форма;
submitButtonElement — кнопка отправки;
errorsElement — контейнер для ошибок.
При сабмите формы останавливается стандартное поведение и генерируется событие submitName.

Методы:
setError(validated) — принимает массив результатов валидации. Извлекает только текстовые ошибки и выводит их в блок ошибок.
setValid(valid) — включает или выключает кнопку отправки формы в зависимости от значения valid.

#### Класс FormContacts

Компонент формы контактных данных. Наследует FormComponent и добавляет работу с полями email и phone.

Интерфейс данных:
IFormContacts включает поля email, phone и объект validated, содержащий результаты валидации каждого поля.

Конструктор:
FormContacts(container, events) — принимает DOM-элемент формы и брокер событий.
Инициализирует поля ввода email и phone.
На каждое поле навешивает обработчик input, который генерирует событие form:contacts-change с переданным значением.

Метод render(data):
Обновляет содержимое полей email и phone, если они переданы в data.
Если переданы validated, показывает ошибки через setError.
Если передано valid, включает или выключает кнопку отправки через setValid.
Возвращает корневой DOM-элемент формы.

#### Класс FormPayment

Компонент формы выбора способа оплаты и ввода адреса. Наследует FormComponent и добавляет управляющие элементы для двух вариантов оплаты.

Интерфейс данных:
IFormPayment содержит поля payment, address и validated — объект результатов валидации каждого поля.

Конструктор:
FormPayment(container, events) — принимает DOM-элемент формы и брокер событий.
Инициализирует элементы:
– поле ввода адреса;
– кнопки выбора оплаты (card, cash).
Навешивает обработчики:
– input для адреса — генерирует событие form:payment-change с новым значением;
– click для каждой кнопки оплаты — генерирует событие form:payment-change с выбранным способом.

Метод setPayment(payment):
Сбрасывает активные стили у обеих кнопок и подсвечивает выбранную в зависимости от значения payment.

Метод render(data):
Обновляет поле адреса и активное состояние кнопки оплаты.
Если передан validated — выводит текст ошибок через setError.
Если передан valid — включает или выключает кнопку отправки через setValid.
Возвращает корневой DOM-элемент формы.

#### Класс Basket

Компонент корзины. Управляет списком товаров в корзине, отображением итоговой суммы и обработкой перехода к оформлению заказа.

Элементы:
– listElements — контейнер списка товаров (.basket**list);
– totalElements — элемент отображения итоговой суммы (.basket**price);
– orderButton — кнопка оформления заказа (.basket\_\_button).

Конструктор:
Basket(container, events) — принимает DOM-элемент корзины и брокер событий.
Инициализирует DOM-элементы и навешивает обработчик на кнопку, генерирующий событие basket:order.

Сеттер items:
Принимает массив DOM-элементов карточек корзины и полностью заменяет содержимое списка.
Также включает или отключает кнопку оформления заказа (кнопка неактивна, если корзина пуста).

Сеттер total:
Устанавливает отображаемую итоговую сумму в формате <число> синапсов.

#### Класс Success

Компонент экрана успешного оформления заказа. Показывает итоговую сумму и предоставляет кнопку закрытия.

Интерфейс данных:
ISuccess содержит поле total — сумма заказа.

Конструктор:
Success(container, events) — принимает DOM-элемент и брокер событий.
Инициализирует:
– descriptionElement (.order-success**description) — текст с итоговой суммой;
– closeButton (.order-success**close) — кнопка закрытия экрана.
При клике на кнопку генерируется событие success:success-close.

Метод render(data):
Если передан total, отображает текст вида “Списано <total> синапсов”.
Возвращает корневой DOM-элемент компонента.

### Презентер

Презентер — центральный слой приложения (реализован в main.ts), который связывает модели и компоненты интерфейса.
Он не хранит данные и не изменяет DOM напрямую: вся работа идёт через модели и View-компоненты.
Презентер только подписывается на события и в ответ вызывает методы моделей и представлений.

1. Управление жизненным циклом приложения

При инициализации:
создаются экземпляры моделей: Items, Cart, Customer;
создаются View-компоненты: Header, Gallery, Modal, CardPreview, Basket, FormPayment, FormContacts, Success;
выполняется загрузка каталога с сервера через Api и Communicator (clientApi.getProducts()).

После загрузки:
данные сохраняются в productsModel (Items);
презентер генерирует событие gallery:updated, которое приводит к отрисовке каталога.

2. Обработка событий моделей

Модели генерируют события, на которые подписан презентер:

gallery:updated — модель Items обновила каталог;
презентер вызывает renderCatalog(), который:
получает список товаров из productsModel;
создаёт карточки через CardCatalog;
передаёт их в gallery.catalog.

cart:changed — модель Cart изменилась (товар добавлен/удалён, корзина очищена);
презентер вызывает renderCart(), который:
строит список элементов корзины через CardBasket;
обновляет компонент Basket;
обновляет счётчик в Header.

customer:changed — модель Customer обновила данные покупателя;
презентер вызывает renderFormPayment() и renderFormContacts(), чтобы:
подтянуть актуальные значения полей;
пересчитать валидацию;
обновить состояние кнопок.

3. Обработка событий представлений

View-компоненты генерируют события, на которые презентер реагирует:

выбор товара в каталоге (gallery:picked);
добавление/удаление товара из корзины (card:add-to-cart, basket:remove-card);
открытие корзины (basket:open);
изменение полей форм (form:payment-change, form:contacts-change);
отправка форм (form:payment-submit, form:contacts-submit);
закрытие экрана успешного заказа (success:success-close).

В ответ презентер:

меняет состояние моделей (Items, Cart, Customer);
открывает/закрывает модальные окна;
инициирует перерисовку нужных View-компонентов.

4. Управление модальными окнами

За модальные окна отвечает компонент Modal, а презентер решает, что в них показывать и когда:
превью товара (CardPreview) — при выборе товара в каталоге;
корзину (Basket) — при клике по кнопке в шапке;
форму оплаты (FormPayment) — при переходе к оформлению;
форму контактов (FormContacts) — после успешной валидации оплаты;
экран успешного заказа (Success) — после удачной отправки заказа.

Используются методы:
modal.open(viewElement) — открыть модалку с указанным содержимым;
modal.close() — закрыть модалку.

5. Взаимодействие с сервером

Работа с сервером сосредоточена в презентере через Communicator:
загрузка каталога: clientApi.getProducts() → данные уходят в productsModel.setItems() → gallery:updated;
отправка заказа: clientApi.sendOrder(order) в обработчике события form:contacts-submit.

После успешной отправки заказа:
очищается корзина (cartModel.deleteAll());
очищаются данные покупателя (customerModel.clearData());
открывается модалка с компонентом Success.
Ошибки запросов логируются в консоль (Error loading data, Error submitting data).

6. Презентер не хранит данные

Все данные живут в моделях:
Items — полный каталог товаров и активный товар;
Cart — список id товаров в корзине, подсчёт суммы и количества;
Customer — способ оплаты, адрес, email и телефон, а также их валидация.

Презентер только:

вызывает методы моделей (setItems, setActiveItem, addItem, deleteItem, addData, validateData, и т.д.);
запрашивает из моделей данные;
передаёт эти данные в методы рендера View-компонентов (render, сеттеры и т.п.).

7. Презентер не изменяет DOM

Всё взаимодействие с DOM инкапсулировано в компонентах View:
Gallery, CardCatalog, CardPreview;
Basket, CardBasket;
FormPayment, FormContacts;
Header, Modal, Success.

Презентер только:

передаёт массив карточек в gallery.catalog;
задаёт basket.items, basket.total;
вызывает formPayment.render(...), formContacts.render(...);
открывает и закрывает модальные окна через Modal.

8. Перерисовка представлений

View-компоненты перерисовываются только:
при событиях от моделей (gallery:updated, cart:changed, customer:changed);
при открытии модального окна (перед modal.open() вызывается соответствующий render).
Это позволяет держать логику обновления интерфейса простой и предсказуемой.

9. Логика корзины

Презентер реализует переключение состояния товара «в корзине / не в корзине» — toggle-логику:
событие card:add-to-cart:
получает активный товар из productsModel;
если товар бесценный (price === null), просто игнорирует;
если товар уже в корзине — удаляет его (cartModel.deleteItem(item)), иначе — добавляет (cartModel.addItem(item));
закрывает модалку превью.

событие basket:remove-card:
удаляет указанный товар из корзины.

После этих операций модель Cart генерирует cart:changed, и презентер обновляет корзину и шапку.

10. Логика оформления заказа

Презентер управляет полным циклом оформления заказа:
событие basket:order:
открывает форму оплаты (FormPayment) и рендерит её с текущими данными.
событие form:payment-change:
обновляет модель Customer (поле address или payment).
событие form:payment-submit:
валидирует данные оплаты;
если есть ошибки — перерисовывает FormPayment с текстами ошибок и выключенной кнопкой;
если всё корректно — открывает форму контактов.

событие form:contacts-change:
обновляет модель Customer (поле email или phone).

событие form:contacts-submit:
валидирует email и телефон;
если есть ошибки — перерисовывает FormContacts с ошибками и выключенной кнопкой;
если всё корректно — собирает объект заказа IOrder и отправляет его на сервер;

после успешной отправки очищает модели и открывает Success.

11. Список событий, которые слушает презентер

Ниже перечислены все события, на которые подписывается презентер (broker.on(...)), и реакция на них:

gallery:updated
Источник: модель Items.
Действие: вызов renderCatalog() → построение карточек и передача их в gallery.catalog.

gallery:picked (слушатель №1)
Источник: CardCatalog (клик по карточке).
Действие:
получает активный товар через productsModel.getActiveItem();
рендерит превью CardPreview;
открывает модалку с превью через modal.open(view).

gallery:picked (слушатель №2, с payload { id: string })
Источник: CardCatalog (клик по карточке, передаётся id).
Действие:
устанавливает активный товар в productsModel.setActiveItem(id).

cart:changed
Источник: модель Cart.
Действие:
вызов renderCart() → обновление Basket;
обновление header.counter (количество товаров).

customer:changed
Источник: модель Customer.
Действие:
renderFormPayment() — обновление формы оплаты;
renderFormContacts() — обновление формы контактов.

basket:open
Источник: Header (клик по кнопке корзины).
Действие:
открытие модального окна с содержимым корзины: modal.open(basket.render()).

card:add-to-cart
Источник: CardPreview (клик по кнопке «В корзину» / «Удалить из корзины»).
Действие:
получает активный товар;
добавляет или удаляет его из Cart;
закрывает модалку.

basket:remove-card
Источник: CardBasket (кнопка удаления конкретной позиции).
Действие:
удаляет товар из корзины через cartModel.deleteItem(data).

basket:order
Источник: Basket (кнопка «Оформить»).
Действие:
рендерит форму оплаты;
открывает её в модалке.

form:payment-change
Источник: FormPayment (input адреса и кнопки выбора оплаты).
Действие:
обновляет модель Customer через customerModel.addData({ [field]: value }).

form:payment-submit
Источник: FormPayment (submit формы).
Действие:
валидирует поля payment и address;
при ошибках — перерисовывает форму с ошибками и выключенной кнопкой;
при успехе — открывает форму контактов.

form:contacts-change
Источник: FormContacts (input email и телефона).
Действие:
обновляет модель Customer через customerModel.addData({ [field]: value }).

form:contacts-submit (слушатель №1)
Источник: FormContacts (submit формы).
Действие:
валидирует email и телефон;
при ошибках — перерисовывает форму с ошибками и выключенной кнопкой.

form:contacts-submit (слушатель №2)
Источник: тот же submit формы контактов.
Действие:
при корректных данных — формирует IOrder, отправляет его через clientApi.sendOrder(order);
по успешному ответу — очищает Cart и Customer, открывает Success.

success:success-close
Источник: Success (кнопка закрытия).
Действие:
закрывает модальное окно через modal.close().
